# 微信联系人分组管理插件

基于 **WAuxiliary** 框架的微信联系人分组管理插件，支持将联系人归类到自定义分组，提供完整的分组 CRUD 操作与批量添加功能。

---

## 触发方式

插件提供以下两种触发入口：

* **长按右上角按钮**：在聊天界面长按右上角“更多功能 / 设置 / 聊天信息”按钮。
* **发送关键词**：在任意聊天框输入 `分组管理` 并点击发送。

---

## 功能概览

### 1. 分组选择页面
* **概览展示**：展示所有分组及各分组的成员数量。
* **排序调整**：支持“上移 / 下移”操作，灵活调整分组显示顺序。
* **快速加入**：点击“选择”可将当前聊天对象直接加入该分组（支持智能去重）。
* **详情管理**：点击分组行即可进入该分组的详情页面。
* **新建分组**：支持创建自定义分组。
* **头像同步**：点击“🔄 同步”按钮手动触发增量同步（包括清理孤立头像、下载新增头像、检测 URL 变化）。

### 2. 分组详情页面
* **基本信息**：编辑分组名称，切换分组启用状态（开关）。
* **成员管理**：查看带头像的成员列表，支持将指定成员“移出”分组。
* **批量操作**：进入批量添加好友页面。
* **安全删除**：提供带确认弹窗的删除操作，防止误删。

### 3. 批量添加好友页面
* **智能过滤**：自动过滤已在当前分组中的好友。
* **多 Tab 分类筛选**：
    * **全部**：包含好友、群聊、订阅号和服务号（带分类标题）。
    * **好友**：仅显示个人好友。
    * **群聊**：仅显示群聊。
    * **订阅号**：从微信数据库读取的订阅号（基于 verifyFlag 条件）。
    * **服务号**：从微信数据库读取的服务号。
    * **微信标签**：每个标签生成独立 Tab，仅显示标签内的联系人。
* **搜索与选择**：支持按昵称或 wxid 搜索；支持“全选 / 反选”快捷操作。
* **提交逻辑**：点击“确定添加”后关闭页面；点击“保存”则留在当前页继续选择。
* **性能优化**：
    * **分批加载**：首屏加载 8 条，后续 50 条，确保滑动流畅。
    * **预渲染机制**：打开对话框后后台自动预加载其他 Tab 内容，实现秒级切换。

## 数据说明

### 数据文件路径
插件支持多开，文件路径将根据当前微信用户自动适配：
```
{WAuxiliary目录}/Resource/Group/groupItems.json
```

### 默认分组配置
若检测到没有相关配置文件，将自动生成以下初始分组：
```json
[
  { "type": "all", "order": 0, "title": "全部", "enable": true, "idList": [] },
  { "type": "group", "order": 1, "title": "群聊", "enable": true, "idList": [] },
  { "type": "friend", "order": 2, "title": "好友", "enable": true, "idList": [] },
  { "type": "official", "order": 3, "title": "官方", "enable": true, "idList": [] },
  { "type": "custom", "order": 4, "title": "示例", "enable": true, "idList": [] }
]
```

### 数据字段结构
* **title (String)**：分组名称，全局唯一。
* **order (int)**：排列顺序，数值越小越靠前。
* **type (String)**：自定义Tab（长按custom的Tab可以显示添加页面）。
* **enable (boolean)**：该分组是否处于启用状态。
* **idList (Array)**：存储分组内成员 wxid 的列表。

## 界面与技术特性

* **外观适配**：自动适配系统的深色与浅色模式。
* **头像系统**：
    * 采用“内存缓存（128张）+ 磁盘缓存”双级缓存方案。
    * 异步下载，并发限制为 8 个，平衡加载速度与系统负载。
* **性能保障**：
    * **缓存复用**：好友昵称 5 分钟内不重复请求，提升响应速度。
    * **索引预处理**：首次加载时后台生成搜索索引，提升检索效率。
    * **线程隔离**：所有文件 IO 操作均在子线程进行，杜绝界面卡顿。

---

## 注意事项

* 分组名称具有唯一性，不可重复。
* 同一 wxid 在同一分组内具有去重机制，不会重复保存。
* 卸载插件时会自动解除 Hook 逻辑，不会对微信原生功能造成影响。
